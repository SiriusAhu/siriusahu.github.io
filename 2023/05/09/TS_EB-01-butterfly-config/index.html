Butterfly主题文档
Butterfly主题文档-主题配置1

1. 你需要安装渲染器

Butterfly主题需要安装hexo-renderer-pug和hexo-renderer-sass渲染器，否则无法正常使用！

2. 配置文件的优先级

每一个主题都有自己的配置文件，如Butterfly主题的配置文件是_config.butterfly.yml。

按Butterfly官方文档的推荐，我们可以把这个配置文件放在和_config.yml同级的目录下。

因为主题的配置文件的优先级高于_config.yml，我们进行的主题配置如果和_config.yml中的配置冲突，那么Hexo会优先使用主题配置文件中的配置。

3.4.3 顶部图

顶部图即网站首页（包括文章页面）顶部的大图。

这里直接引用官方的解释：

官方文档：
如果不要显示顶部图，可直接配置 disable_top_img: true

顶部图的获取顺序，如果都没有配置，则不显示顶部图。
页面顶部图的获取顺序：
各自配置的 top_img > 配置文件的 default_top_img
文章页顶部图的获取顺序：
各自配置的 top_img > cover > 配置文件的 default_top_img

配置中的值：（注意：tag_img系列和category_img系列的配置是针对子页面的，并非主页）

配置 解释
index_img 主页的 top_img
default_top_img 默认的 top_img，当页面的 top_img 没有配置时，会显示 default_top_img
archive_img 归档页面的 top_img
tag_img tag 子页面 的 默认 top_img
tag_per_img tag 子页面的 top_img，可配置每个 tag 的 top_img
category_img category 子页面 的 默认 top_img
category_per_img category 子页面的 top_img，可配置每个 category 的 top_img

这里的配置参数接受：

  1. 留空：即显示默认的顶部图
  2. 图片链接：网络链接或本地路径（/img/xxxx.png）
  3. transparent：透明
  4. false：不显示顶部图
  5. 颜色：包括
    • HEX：'#000000' *需要用引号包裹
    • RGB：rgb(255, 255, 255) *需写全
    • 单词：red、aqua等
    • 渐变色：'linear-gradient(20deg, #0062be, #925696, #cc426e, #fb0347)' *需要用引号包裹（<-这个颜色还挺好看的）

这部分还有一些其他的配置，可以参考官方文档。

  • 如果又想方便又想美观的话，可以直接配置index_img和default_top_img即可。
